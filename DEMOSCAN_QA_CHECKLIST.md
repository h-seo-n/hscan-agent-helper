# DemoScan Step Type Manual QA Checklist

시작 전 준비:

- `pnpm dev:be`, `pnpm dev:ext`, `pnpm dev:demo`를 실행한다.
- Chrome에서 확장 프로그램을 로드한 뒤 DemoScan `http://localhost:5174`를 연다.
- Hscan 사이드패널을 열고 각 항목의 시작 화면에서 사용자 발화를 입력한다.

| Step type | 시작 화면 | 사용자 발화 | 기대 결과 | 시각 피드백 여부 |
| --- | --- | --- | --- | --- |
| `navigate` | 홈 `/` | `내 영상 다운로드 받고 싶어` | `내 영상 목록` 탭 또는 진입점을 통해 `/images`로 이동한다. 이동 후 새 snapshot을 수집하고 다운로드 관련 위치를 다시 안내한다. | 있음. 이동 전 대상 요소가 하이라이트된다. |
| `highlight` | 영상 목록 `/images` | `다운로드 어디서 해?` | 하단 작업바의 `다운로드` 버튼이 안내 대상이 되고, 버튼 위치를 알려준다. | 있음. `다운로드` 버튼에 하이라이트 오버레이가 표시된다. |
| `click` | 홈 `/` 또는 임의 화면 | `내 정보 탭 눌러줘` | `내 정보` 탭을 클릭 실행하고 `/my` 화면으로 이동한다. step 상태는 `done`으로 처리된다. | 있음. 클릭 전 `내 정보` 탭이 잠깐 하이라이트된다. |
| `input` | CD 배송 신청 `/cd-request` | `CD 신청하고 싶어` | `수령인 이름` 입력칸에 포커스가 가고 사용자가 입력할 때까지 대기한다. 사용자가 값을 입력하면 step이 완료된다. | 있음. 입력칸이 하이라이트된다. |
| `scroll` | 홈 `/` | `고객센터 위치로 스크롤해서 보여줘` | `고객센터` 링크 위치로 스크롤하고 해당 위치를 보여준다. | 있음. 스크롤 대상 링크가 하이라이트된다. |
| `explain` | 홈 `/` | `이 화면에서 뭘 할 수 있어?` | 클릭, 입력, 이동 없이 사이드바에 설명성 응답이 표시된다. 대상 요소가 없거나 일반 설명이면 explain step으로 완료된다. | 없음. `explain` 자체는 DOM 하이라이트 없이 텍스트 설명만 표시된다. |

## 확인 메모

- `highlight` 요청은 현재 구현상 `highlight + explain` 조합으로 나올 수 있다.
- `input` step은 자동 입력하지 않고 입력 위치를 안내한 뒤 사용자 입력을 기다린다.
- `click`, `navigate`, `scroll`, `input`은 실행 전 또는 실행 중 대상 요소에 하이라이트를 표시한다.
- `explain`은 화면 요소 피드백이 아니라 사이드바 텍스트 응답으로 확인한다.
